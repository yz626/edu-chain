## 前置条件

### 1. 环境要求

- **Go版本**: 1.18+
- **FISCO BCOS**: 2.0+
- **操作系统**: Linux (推荐Ubuntu 20.04+) 或 macOS

### 2. 安装Go依赖

```bash
# 在项目根目录执行
cd e:/Code/Go/edu-chain

# 下载FISCO BCOS Go SDK依赖
go get github.com/FISCO-BCOS/go-sdk@latest

# 确保依赖完整
go mod tidy
```

### 3. 防火墙配置

确保以下端口可访问：
- `8545/8546`: RPC服务端口
- `30300/30301`: P2P服务端口

## 快速开始

### 步骤1: 启动FISCO BCOS节点

```bash
# 方式A: 使用Docker快速启动（开发环境）
docker run -d --name fisco-bcos \
  -v ~/fisco/nodes:/data \
  -p 8545:8545 \
  -p 30300:30300 \
  fiscoorg/fisco-bcos:latest

# 方式B: 使用官方构建脚本（生产环境）
# 参考: https://fisco-bcos-documentation.readthedocs.io
```

### 步骤2: 配置项目

```bash
# 复制配置示例
cp config/config.yaml.example config/config.yaml

# 编辑配置文件
vim config/config.yaml
```

### 步骤3: 初始化区块链客户端

```go
// 在您的服务初始化代码中
import (
    "github.com/yz626/edu-chain/config"
    "github.com/yz626/edu-chain/internal/blockchain/fiscobcos"
)

func initBlockchain() (*fiscobcos.Client, error) {
    cfg, err := config.Load()
    if err != nil {
        return nil, err
    }
    
    return fiscobcos.NewClient(&cfg.Blockchain.FISCOBCOS)
}
```

### 步骤4: 使用区块链功能

```go
// 发行证书
txHash, err := client.IssueCertificate(ctx, fiscobcos.CertificateData{
    CertID:      "CERT-2024-001",
    StudentID:   "2021001",
    StudentName: "张三",
    CourseName:  "区块链技术",
    CourseScore: "95",
    Issuer:      "教育联盟",
    IssueDate:   "2024-01-15",
})

// 验证证书
isValid, err := client.VerifyCertificate(ctx, "CERT-2024-001")
```

## FISCO BCOS节点部署

### 1. 单机部署（开发/测试）

```bash
# 创建工作目录
mkdir -p ~/fisco && cd ~/fisco

# 下载构建脚本
curl -LO https://github.com/FISCO-BCOS/FISCO-BCOS/releases/download/v2.9.1/build_chain.sh
chmod +x build_chain.sh

# 生成节点配置（4个节点，端口默认）
./build_chain.sh -l 127.0.0.1:4 -p 30300,8545

# 启动节点
cd nodes/127.0.0.1
bash start_all.sh

# 检查节点状态
bash nodes/127.0.0.1/script/health_check.sh
```

### 2. 多机部署（生产）

```bash
# 节点1 (192.168.1.10)
./build_chain.sh -l 192.168.1.10:2 -p 30300,8545 -e fisco-bcos

# 节点2 (192.168.1.11)
./build_chain.sh -l 192.168.1.11:2 -p 30300,8545 -e fisco-bcos

# 配置并启动各节点...
```

### 3. 获取SDK连接凭证

```bash
# 查看节点配置
cat nodes/127.0.0.1/node0/config.ini

# 获取SDK连接信息
# NodeUrl: 127.0.0.1:8545
# 群组ID: 1
```

## 核心功能实现

### 1. 初始化区块链客户端

```go
// internal/blockchain/fiscobcos/client.go

package fiscobcos

import (
    "github.com/yz626/edu-chain/config"
)

func NewBlockchainClient(cfg *config.Config) (*Client, error) {
    if cfg.Blockchain.Type != "fiscobcos" {
        return nil, fmt.Errorf("不支持的区块链类型: %s", cfg.Blockchain.Type)
    }
    
    return NewClient(&cfg.Blockchain.FISCOBCOS)
}
```

### 2. 证书上链服务

```go
// internal/service/certificate.go

package service

type CertificateService struct {
    blockchainClient *fiscobcos.Client
    db               *gorm.DB
}

func (s *CertificateService) IssueCertificate(req *IssueCertRequest) (*Certificate, error) {
    // 1. 生成证书ID
    certID := generateCertID()
    
    // 2. 构造证书数据
    certData := fiscobcos.CertificateData{
        CertID:      certID,
        StudentID:   req.StudentID,
        StudentName: req.StudentName,
        CourseName:  req.CourseName,
        CourseScore: req.CourseScore,
        Issuer:      req.Issuer,
        IssueDate:   req.IssueDate,
    }
    
    // 3. 上链
    txHash, err := s.blockchainClient.IssueCertificate(context.Background(), certData)
    if err != nil {
        return nil, fmt.Errorf("上链失败: %w", err)
    }
    
    // 4. 存储到本地数据库
    cert := &Certificate{
        ID:          certID,
        TxHash:      txHash,
        Status:      "issued",
        CreatedAt:   time.Now(),
    }
    
    if err := s.db.Create(cert).Error; err != nil {
        return nil, err
    }
    
    return cert, nil
}
```

### 3. 证书验证服务

```go
func (s *CertificateService) VerifyCertificate(certID string) (*VerificationResult, error) {
    // 1. 链上验证
    isValid, err := s.blockchainClient.VerifyCertificate(context.Background(), certID)
    if err != nil {
        return nil, fmt.Errorf("链上验证失败: %w", err)
    }
    
    // 2. 查询链上详情
    certData, err := s.blockchainClient.QueryCertificate(context.Background(), certID)
    if err != nil {
        return nil, fmt.Errorf("查询链上数据失败: %w", err)
    }
    
    return &VerificationResult{
        IsValid:     isValid,
        Certificate: certData,
    }, nil
}
```

## 国密版本配置

### 1. 使用国密算法

FISCO BCOS支持国密算法（SM2/SM3/SM4），配置如下：

```yaml
blockchain:
  fiscobcos:
    # 国密版本配置
    gm: true
    
    # 国密账户
    gm_account_key: "gm-private-key-hex"
    gm_account_cert: "./certs/gm-node.crt"
    gm_account_key_file: "./certs/gm-node.key"
    
    # 国密SSL连接
    gm_ssl_ca: "./certs/gmca.crt"
    gm_ssl_cert: "./certs/gmnode.crt"
    gm_ssl_key: "./certs/gmnode.key"
```

### 2. 编译国密版本

```bash
# 编译国密版FISCO BCOS
./build.sh -DGM

# 编译国密版SDK
go build -tags=gmsm .
```

## 常见问题

### Q1: 连接节点失败

**问题**: `connect failed: dial tcp 127.0.0.1:8545: connect: connection refused`

**解决**:
1. 检查FISCO BCOS节点是否启动
2. 确认节点RPC服务端口配置正确
3. 检查防火墙是否放行端口

```bash
# 检查节点进程
ps aux | grep fisco-bcos

# 检查端口监听
netstat -tlnp | grep 8545
```

### Q2: 交易超时

**问题**: `transaction timeout`

**解决**:
1. 增加超时配置
2. 检查网络连接
3. 确认节点共识正常

```yaml
blockchain:
  fiscobcos:
    timeout: 60  # 增加超时时间
```

### Q3: 私钥格式错误

**问题**: `invalid private key format`

**解决**:
1. 确认私钥是64位十六进制字符串（不带0x前缀）
2. 检查私钥文件编码是否为UTF-8

### Q4: 合约调用失败

**问题**: `call contract failed: contract not found`

**解决**:
1. 确认合约已正确部署
2. 检查合约地址配置正确
3. 确认ABI JSON格式正确
