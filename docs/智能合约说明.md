# CertificateRegistry 智能合约说明

## 一、概述

`CertificateRegistry` 是 EduChain 系统的核心链上存证合约，部署于 Ethereum 兼容链（支持 FISCO BCOS EVM 模式）。

**设计原则**：
- 链上只存核心数据（证书哈希 + 状态），详细业务数据存链下 MySQL
- 通过 `certId`（链下 UUID 转 `bytes32`）关联链上与链下记录
- 不引入 NFT/Token，不做链上分页，结构简单、可直接部署使用

---

## 二、角色说明

| 角色 | 说明 | 权限 |
|------|------|------|
| `owner` | 合约部署者（监管方/教育部门） | 管理颁发机构、暂停/恢复合约、转移所有权 |
| `issuer` | 授权的颁发机构（高校） | 颁发/撤销/恢复证书 |
| 任何人 | — | 查询、验证证书（只读） |

---

## 三、数据结构

### CertRecord（链上证书记录）

| 字段 | 类型 | 说明 |
|------|------|------|
| `certHash` | `bytes32` | 证书完整数据的 SHA-256 哈希，用于链下完整性校验 |
| `issuer` | `address` | 颁发机构区块链地址 |
| `issuedAt` | `uint64` | 颁发时间戳（Unix 秒） |
| `revoked` | `bool` | 是否已撤销 |
| `revokedAt` | `uint64` | 撤销时间戳（0 = 未撤销） |
| `revokeReason` | `string` | 撤销原因简述 |

### IssuerInfo（颁发机构信息）

| 字段 | 类型 | 说明 |
|------|------|------|
| `name` | `string` | 机构名称 |
| `authorized` | `bool` | 当前是否处于授权状态 |
| `authorizedAt` | `uint64` | 最近一次授权时间戳 |

## 四、函数接口

### 4.1 Owner 管理

```solidity
// 查询合约所有者
function owner() external view returns (address)

// 转移所有权（新地址不可为零地址）
function transferOwnership(address newOwner) external
```

### 4.2 紧急暂停

```solidity
// 暂停合约（仅 owner）
function pause() external

// 恢复合约（仅 owner）
function unpause() external

// 查询暂停状态
function paused() external view returns (bool)
```

### 4.3 颁发机构管理（仅 owner）

```solidity
// 添加或重新授权机构
function addIssuer(address issuer, string calldata name) external

// 批量添加授权机构
function addIssuerBatch(address[] calldata issuers, string[] calldata names) external

// 撤销机构授权（历史证书不受影响）
function revokeIssuer(address issuer) external

// 查询机构授权信息
function getIssuerInfo(address issuer)
    external view
    returns (bool authorized, string memory name, uint64 authorizedAt)
```

### 4.4 证书颁发（仅授权机构）

```solidity
// 单张证书上链存证
function issueCertificate(bytes32 certId, bytes32 certHash) external

// 批量证书上链存证
function issueCertificateBatch(
    bytes32[] calldata certIds,
    bytes32[] calldata certHashes
) external
```

**certId 生成规则**：将链下 UUID 字符串转为 bytes32：
```go
// Go 侧示例
certId := crypto.Keccak256Hash([]byte(uuidString))
```

**certHash 生成规则**：将证书完整 JSON 数据做 SHA-256：
```go
// Go 侧示例
hash := sha256.Sum256(certJsonBytes) // [32]byte，直接对应 bytes32
```

### 4.5 证书撤销与恢复（仅原颁发机构）

```solidity
// 撤销证书（只有原颁发机构可操作）
function revokeCertificate(bytes32 certId, string calldata reason) external

// 恢复被撤销的证书
function restoreCertificate(bytes32 certId) external
```

### 4.6 查询与验证（任何人可调用，无 gas）

```solidity
// 查询证书是否存在
function certExists(bytes32 certId) external view returns (bool)

// 获取证书完整链上记录
function getCertificate(bytes32 certId)
    external view
    returns (
        bytes32 certHash,
        address issuer,
        uint64  issuedAt,
        bool    revoked,
        uint64  revokedAt,
        string memory revokeReason
    )

// 验证证书（哈希匹配 + 未撤销）
function verifyCertificate(bytes32 certId, bytes32 certHash)
    external view
    returns (bool valid, bool revoked)

// 批量验证证书
function verifyCertificateBatch(
    bytes32[] calldata certIds,
    bytes32[] calldata certHashes
) external view returns (bool[] memory valids, bool[] memory revokeds)

// 获取全局统计
function getStats() external view returns (uint256 totalIssued, uint256 totalRevoked)
```

## 五、事件

| 事件 | 触发时机 |
|------|----------|
| `OwnershipTransferred(prev, newOwner)` | 所有权转移 |
| `Paused(by)` | 合约暂停 |
| `Unpaused(by)` | 合约恢复 |
| `IssuerAuthorized(issuer, name, ts)` | 机构被授权 |
| `IssuerRevoked(issuer, ts)` | 机构授权被撤销 |
| `CertificateIssued(certId, certHash, issuer, ts)` | 证书上链 |
| `CertificateRevoked(certId, reason, by, ts)` | 证书被撤销 |
| `CertificateRestored(certId, by, ts)` | 证书被恢复 |
| `BatchIssued(issuer, count, ts)` | 批量颁发完成 |

## 六、自定义错误

| 错误 | 说明 |
|------|------|
| `Unauthorized()` | 调用者无权限 |
| `ZeroAddress()` | 传入了零地址 |
| `EmptyParam()` | 必填参数为空（bytes32(0) 或空字符串） |
| `BatchEmpty()` | 批量操作数组长度为 0 |
| `ArrayLengthMismatch()` | 两个并行数组长度不一致 |
| `ContractPaused()` | 合约已暂停，写操作被拒绝 |
| `ContractNotPaused()` | 合约未暂停（unpause 前置检查） |
| `IssuerNotAuthorized(issuer)` | 机构未被授权 |
| `CertAlreadyExists(certId)` | 证书 ID 已存在，重复颁发 |
| `CertNotFound(certId)` | 证书不存在 |
| `CertAlreadyRevoked(certId)` | 证书已撤销，不可重复撤销 |
| `CertNotRevoked(certId)` | 证书未被撤销，不可执行恢复 |
| `NotCertIssuer(certId)` | 调用者非该证书的原颁发机构 |

## 七、部署说明

### 编译

```bash
# 使用 solc 编译（要求 >= 0.8.19）
solc --abi --bin --optimize contracts/CertificateRegistry.sol -o bin/contracts/
```

### 部署参数

构造函数需传入 `ownerName`（部署机构名称），部署者自动成为 owner 并获得颁发机构资格。

```javascript
// 部署示例（ethers.js）
const factory = new ethers.ContractFactory(abi, bytecode, signer);
const contract = await factory.deploy("教育部学历认证中心");
await contract.deployed();
console.log("合约地址:", contract.address);
```

### Go 侧调用示例

```go
// 1. 生成 certId（UUID -> bytes32）
certId := crypto.Keccak256Hash([]byte(certificate.ID)) // [32]byte

// 2. 生成 certHash（证书 JSON -> SHA-256）
certJSON, _ := json.Marshal(certData)
hashArr := sha256.Sum256(certJSON)  // [32]byte

// 3. 调用合约
tx, err := contract.IssueCertificate(auth, certId, hashArr)
```

## 八、链下数据关联

```
链下 MySQL (certificates 表)
┌─────────────────────────────────────────┐
│ id (UUID)          ← 原始 certId        │
│ blockchain_cert_hash ← certHash (hex)  │
│ blockchain_tx_hash   ← 交易哈希         │
│ on_chain_status      ← 上链状态         │
│ name, major, ...     ← 详细业务数据     │
└─────────────────────────────────────────┘
           │  keccak256(id) = certId
           ▼
链上 Solidity (_certs mapping)
┌─────────────────────────────────────────┐
│ certId  (bytes32) ← keccak256(UUID)     │
│ certHash (bytes32) ← sha256(certJSON)   │
│ issuer, issuedAt, revoked, ...          │
└─────────────────────────────────────────┘
```

验证流程：
1. 用户提供证书编号（或扫码）
2. 后端从 MySQL 读取证书数据，重新计算 `certHash`
3. 调用合约 `verifyCertificate(certId, certHash)` → 返回 `valid=true` 即为真实有效证书
