## 二、项目开发流程规划

### 2.1 总体开发策略

采用 **"自底向上、分层实现、迭代开发"** 的策略：

```
┌─────────────────────────────────────────────────────────┐
│                    开发顺序                               │
├─────────────────────────────────────────────────────────┤
│  第1步: 基础设施层 (数据层+工具层)                        │
│    ↓                                                    │
│  第2步: 区块链交互层                                    │
│    ↓                                                    │
│  第3步: 业务逻辑层                                      │
│    ↓                                                    │
│  第4步: 服务接口层                                      │
│    ↓                                                    │
│  第5步: 路由与控制器                                    │
│    ↓                                                    │
│  第6步: 前端集成                                        │
└─────────────────────────────────────────────────────────┘
```

### 2.2 开发阶段详细规划

#### 阶段五：服务接口层 (第9-10周)

**任务清单**:

**API接口设计**:

```go
// 认证模块
POST   /api/v1/auth/login
POST   /api/v1/auth/register
POST   /api/v1/auth/refresh

// 证书模块
POST   /api/v1/certificates          // 创建证书
GET    /api/v1/certificates/:id      // 查询证书
POST   /api/v1/certificates/batch    // 批量创建
POST   /api/v1/certificates/:id/sign // 签名上链

// 验证模块  
POST   /api/v1/verify                // 验证证书
POST   /api/v1/verify/batch          // 批量验证
GET    /api/v1/verify/history        // 验证历史

// 组织模块
POST   /api/v1/organizations         // 创建组织
GET    /api/v1/organizations/:id     // 查询组织
POST   /api/v1/organizations/:id/approve // 审批组织
```

## 四、技术选型确认

### 4.1 当前技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| 后端 | Go 1.24 + Gin | 高性能HTTP框架 |
| 数据库 | MySQL 8.0 | 关系型数据库 |
| 缓存 | Redis 7.x | 会话和热点数据 |
| 区块链 | FISCO BCOS 2.0 | 联盟链 |
| 权限 | Casbin | RBAC权限控制 |
