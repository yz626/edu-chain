# Wire 依赖注入使用指南

## 什么是 Wire？

Wire 是 Google 开发的 Go 语言依赖注入框架，使用**代码生成**的方式实现依赖注入，与运行时反射的 DI 框架（如 dig、fx）不同，Wire 在编译时生成所有依赖初始化代码，没有任何反射开销。

## 核心概念

### 1. Provider

Provider 是返回依赖项的函数。例如：

```go
// NewDB 是一个 Provider，返回 *gorm.DB
func NewDB(cfg *config.Config) (*gorm.DB, error) {
    // 创建数据库连接
}
```

### 2. ProviderSet

ProviderSet 是 Provider 的集合，用于组织相关的 Provider：

```go
var ProviderSet = wire.NewSet(NewDB, NewDBCloser)
```

### 3. Injector

Injector 是 Wire 生成的函数，用于初始化所有依赖：

```go
// 这是 Wire 生成的代码
func InitializeApp(cfg *config.Config) (*AppDependencies, error) {
    // Wire 自动分析依赖图并生成正确的初始化顺序
    db, err := NewDB(cfg)
    if err != nil {
        return nil, err
    }
    // ...
}
```

## 工作原理

```
┌─────────────────────────────────────────────────────────────┐
│                        Wire 工作流程                          │
├─────────────────────────────────────────────────────────────┤
│  1. 定义 Provider 函数                                       │
│     (如 NewDB, NewRedis, NewUserRepository)               │
│                                                              │
│  2. 将 Provider 注册到 ProviderSet                         │
│     var ProviderSet = wire.NewSet(NewDB, ...)              │
│                                                              │
│  3. 运行 wire 生成器                                         │
│     go generate ./wire 或 wire github.com/xxx/wire        │
│                                                              │
│  4. Wire 分析依赖图，生成 wire_gen.go                       │
│     - 静态分析所有 Provider 的参数和返回值                   │
│     - 构建依赖图                                             │
│     - 拓扑排序确定初始化顺序                                 │
│     - 生成初始化代码                                         │
│                                                              │
│  5. 编译运行                                                │
└─────────────────────────────────────────────────────────────┘
```

## 项目中的使用示例

### 定义 Provider

在 `internal/data/db/db.go` 中：

```go
package db

import "github.com/google/wire"

// 定义 Provider 集合
var ProviderSet = wire.NewSet(NewDB, NewDBCloser)

// Provider 函数
func NewDB(cfg *config.Config) (*gorm.DB, error) {
    // 创建数据库连接
}

// Provider 函数 - 清理函数
func NewDBCloser(db *gorm.DB) func() error {
    return func() error {
        sqlDB, _ := db.DB()
        return sqlDB.Close()
    }
}
```

### 使用 ProviderSet

在 `wire/provider.go` 中组合 Provider：

```go
package wire

import (
    "github.com/google/wire"
    "github.com/yz626/edu-chain/internal/data/db"
    // ... 其他导入
)

// 定义完整的应用依赖集合
var AppSet = wire.NewSet(
    db.ProviderSet,              // 导入数据库 Provider
    // redis.ProviderSet,         // 导入 Redis Provider
    // repository.ProviderSet,    // 导入仓储 Provider
    
    // 直接定义 Provider
    NewConfig,
    NewLogger,
    NewHTTPServer,
    
    // 结构体注入
    wire.Struct(new(AppDependencies), "*"),
)
```

### 生成代码

```bash
# 在 wire 目录下运行
cd wire
wire

# 或者使用 go generate
go generate ./wire
```

Wire 会生成 `wire_gen.go` 文件，包含类似以下代码：

```go
// Code generated by Wire. DO NOT EDIT.

func InitializeApp(cfg *config.Config) (*AppDependencies, func(), error) {
    database, err := db.NewDB(cfg)
    if err != nil {
        return nil, nil, err
    }
    
    logger, err := NewLogger(cfg)
    if err != nil {
        return nil, nil, err
    }
    
    redisClient, err := NewRedis(cfg, logger)
    if err != nil {
        return nil, nil, err
    }
    
    // ... 更多初始化
    
    app := &AppDependencies{
        Config:   cfg,
        Logger:   logger,
        DB:       database,
        Redis:    redisClient,
        // ...
    }
    
    return app, cleanup, nil
}
```

## 在项目中使用

### 1. 启动应用

```go
// main.go
package main

import (
    "github.com/yz626/edu-chain/wire"
)

func main() {
    if err := wire.RunApp("config/config.yaml"); err != nil {
        log.Fatal(err)
    }
}
```

### 2. 手动初始化（如果不需要自动生成的代码）

```go
// 使用手动编写的初始化函数
app, cleanup, err := wire.InitApp("config/config.yaml")
if err != nil {
    log.Fatal(err)
}
defer cleanup()

// 启动服务器
app.Server.Start()
```

## 优势

| 优势 | 说明 |
|------|------|
| **编译时检查** | 依赖错误在编译时被发现 |
| **无反射开销** | 生成的代码直接调用 |
| **易于理解** | 生成的代码可读 |
| **更好的 IDE 支持** | 静态分析，代码跳转正常 |
| **适合大型项目** | 清晰的依赖图 |

## 注意事项

1. **Provider 必须是函数**：Wire 只识别函数类型的 Provider
2. **返回值必须明确**：返回类型必须明确，不能是接口（除非使用 Provider 绑定）
3. **依赖顺序自动计算**：不需要关心初始化顺序，Wire 会自动计算
4. **清理函数**：可以使用函数类型返回清理逻辑
